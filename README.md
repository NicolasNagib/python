# Projetos em Python

Coleção de projetos Python desenvolvidos para fins educacionais e práticos, cobrindo diversos tópicos e tecnologias.

## Projetos

### Banco Nagib
Um sistema bancário fictício desenvolvido para estudar e praticar Programação Orientada a Objeto (POO) em Python.

**Características**: Classes de cliente, contas corrente e poupança, operações bancárias básicas.  
[Saiba mais](banco-nagib/README.md)

---

### Dashboard Financeiro
Um dashboard interativo para organização financeira familiar com visualização de dados por gráficos, filtros específicos e relatórios.

**Características**: Visualização de transações, filtros por período, análise gráfica de gastos.  
[Saiba mais](dashboard/README.md)

---

### RPG Eltoria
Um mini-jogo RPG de terminal divertido e interativo, ainda em desenvolvimento. Oferece um sistema completo de combate, progresso de personagem e exploração de mundos.

**Características**: Sistema de classes, combate por turnos, múltiplos mundos, sistema de inventário e progressão.  
[Saiba mais](rpg/README.md)

---

### URL Shortener
Uma aplicação web para encurtar URLs longas de forma rápida e simples, com interface intuitiva e banco de dados persistente.

**Características**: Encurtamento de URLs, aliases personalizados, API REST, interface web amigável.  
[Saiba mais](url-shortener/README.md)

---

### To-Do List
Um gerenciador de tarefas web moderno com interface intuitiva, backend robusto e banco de dados persistente.

**Características**: Adicionar/deletar tarefas, marcar como concluído, avisos de duplicação, interface responsiva.  
[Saiba mais](To-do-list/README.md)

---

### Downloads de Música
Ferramenta para download de músicas de plataformas online.

[Saiba mais](musica/README.md)

---

## Tecnologias Utilizadas

- **Python 3.x**: Linguagem principal
- **Flask**: Web framework
- **Streamlit**: Interface interativa
- **SQLite**: Banco de dados
- **OOP**: Programação Orientada a Objeto
- **HTML/CSS/JavaScript**: Frontend web

## Como Contribuir

Todos os projetos são de código aberto. Contribuições em forma de:
- Relatos de bugs
- Novas funcionalidades
- Melhorias de código
- Documentação

são bem-vindas!
